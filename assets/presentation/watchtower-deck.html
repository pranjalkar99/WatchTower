Hackathon MVP
The Problem

AI agents with real tools are attack surfaces

  • Agents call real tools — file read, webhooks, APIs, database queries
  • Prompt injection hijacks agent intent and chains dangerous tool calls
  • Generic classifiers miss paraphrased attacks and tool-chain abuse

Without protection

An attacker says "ignore previous instructions" → agent reads .env, POSTs secrets to webhook.site — in seconds.

The Solution

SOC-style command center + 3-line SDK

  • Live dashboard — WebSocket-fed security events, attack graph, incident reports
  • Drop-in SDK — wrap any agent with sentinel.protect()
  • Block before tools run — quarantine agent, block DNS egress, halt file reads

Architecture

User prompt → CASCADE detection → allow or block → tool execution → live dashboard updates

→Prompt in → SDK → FastAPI backend → WebSocket → Next.js SOC

Fast offline · Hybrid production · Agent max accuracy

SDK Integration

Protect any agent in 3 lines

from watchtower import SentinelClient sentinel = SentinelClient("http://localhost:8000") result = await sentinel.protect("travel", user_message, run=lambda: graph.invoke(...))
  • pip install -e ./watchtower
  • Works with LangGraph, LangChain, any async agent
  • Blocks before run() executes on attack detection
